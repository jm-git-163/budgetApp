# AGENTS.md

## 프로젝트 설명
이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다.
사용자는 CSV 파일의 거래 데이터를 읽고, 거래 추가와 조회, 통계 계산 같은 가계부 작업을 CLI로 수행한다.

## 코딩 규칙
- 모든 함수와 메서드에는 타입 힌트를 반드시 작성한다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.
- 가능한 한 작은 함수로 나누고, 역할이 겹치면 공통 로직을 분리한다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 새로운 기능은 실패하는 테스트를 먼저 추가한 뒤 구현한다.
- 기존 동작을 바꾸는 경우에도 회귀 테스트를 먼저 보강한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.
- 복잡도가 높아지는 로직은 조건 분기 분리, 조기 반환, 헬퍼 함수 분해로 개선한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 진행한다.
- 품질 검수 결과에서 지적된 문제는 수정한 뒤에만 커밋한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 즉시 커밋한다.
- 기능이 완성되면 푸시까지 진행한다.
- 커밋 메시지는 기능 단위가 드러나도록 작성한다.
